# Onboarding primera semana

Esta guía es para el repo de portafolio personal. El sitio de tareas se lee; este repo se edita.

## Primeros 30 minutos

1. Editá [Acerca de mí](acerca.md).
2. Abrí [Primera entrada](portfolio/01-primera-entrada.md) y reemplazá los textos de ejemplo por tu evidencia real.
3. Revisá [Política AI-safe](politica-uso-ia.md).
4. Abrí [Mapa de evidencias](portfolio/mapa-evidencias.md) y marcá la clase 1 como `Pendiente` o `Mínimo`.
5. Probá `mkdocs serve` o `mkdocs build --strict`.

## Primera semana: resultado mínimo

- Un archivo, URL, captura o commit que pruebe que el portafolio existe.
- Una primera entrada con objetivo, evidencia, límite, reproducibilidad y declaración AI-safe.
- Una nota que explique qué se lee en el sitio de tareas y qué se edita en este repo.
- Una respuesta A0 corta: qué parte podrías defender sin IA.

## Flujo para cada práctica

1. Leé la consigna en el sitio de tareas.
2. Trabajá en notebook, script, repo o carpeta de equipo según corresponda.
3. Guardá outputs mínimos: tabla, gráfico, log, contrato, check o captura.
4. Publicá una entrada breve en este portafolio.
5. Verificá que no publicaste datos privados, `.env`, tokens ni respuestas A0 completas.
6. Entregá en WebAsignatura solo el link o archivo pedido.

## Plantilla mínima por entrada

```text
## Objetivo
## Evidencia
## Reproducibilidad
## Decisión técnica
## Límite o riesgo
## Uso de IA
## Microdefensa
```

## No mezcles

- consignas del sitio de tareas como si fueran evidencia propia;
- evidencia de otra materia como si fuera de esta;
- trabajo de equipo sin explicar tu contribución individual;
- feedback docente privado, notas o datos de compañeros.
